# 通知系统测试指南

## 功能验证清单

### ✅ 已完成的改进
1. **所有通知类型均可点击**
   - TASK (任务通知) - 显示安装日志和进度
   - MESSAGE (消息通知) - 显示详细的消息内容
   - WARNING (警告通知) - 显示警告信息
   - ERROR (错误通知) - 显示错误详情

### 🧪 测试步骤

打开浏览器控制台 (F12)，运行以下测试命令：

#### 1. 测试错误通知
\`\`\`javascript
testNotification.error('数据库连接失败，请检查网络配置。\n\n错误代码: ECONNREFUSED\n主机: localhost:5432')
\`\`\`
**预期结果**: 
- 通知中心显示红色错误卡片
- 点击后打开模态框，显示完整错误信息
- 模态框顶部显示红色渐变图标

#### 2. 测试警告通知
\`\`\`javascript
testNotification.warning('系统检测到磁盘空间不足\n\n当前可用空间: 2.3 GB\n建议释放空间: 至少 5 GB')
\`\`\`
**预期结果**:
- 通知中心显示黄色警告卡片
- 点击后打开模态框，显示警告详情
- 模态框顶部显示黄橙渐变图标

#### 3. 测试消息通知
\`\`\`javascript
testNotification.info('欢迎使用 MAI Launcher v1.0.0\n\n新功能:\n• 全新的通知系统\n• 改进的下载管理\n• 暗黑模式支持')
\`\`\`
**预期结果**:
- 通知中心显示浅蓝色消息卡片
- 点击后打开模态框，显示消息内容
- 模态框顶部显示蓝色渐变图标

#### 4. 测试任务通知
\`\`\`javascript
testNotification.task('MaiBot Instance', 30)
\`\`\`
**预期结果**:
- 通知中心显示白色任务卡片，带进度条
- 点击后打开模态框，显示模拟日志和进度信息
- 模态框顶部显示蓝色渐变终端图标

#### 5. 更新任务进度
\`\`\`javascript
// 先获取任务ID（上一条任务通知返回的ID）
testNotification.updateTask('<task-id>', 75, 'installing', '正在安装 Python 依赖...')
\`\`\`

#### 6. 完整演示
\`\`\`javascript
testNotification.demo()
\`\`\`
**预期结果**: 自动创建所有类型的通知并模拟进度更新

### ✨ 验证要点

#### 通知卡片交互
- [x] 所有类型的通知卡片都应该显示 `cursor-pointer`
- [x] 鼠标悬停时应有缩放效果
- [x] 点击后应打开对应的详情模态框

#### 模态框显示
- [x] MESSAGE/WARNING/ERROR: 显示彩色背景的消息框
- [x] TASK: 显示黑色终端风格的日志区域
- [x] 支持 ESC 键关闭
- [x] 点击遮罩层关闭

#### 视觉反馈
- [x] MESSAGE: 蓝色边框和浅蓝色背景
- [x] WARNING: 黄色边框和浅黄色背景  
- [x] ERROR: 红色边框和浅红色背景
- [x] TASK: 进度条和实时日志

### 🎯 核心改进

**之前**: 只有任务通知可以点击查看详情
**现在**: 所有通知类型都可点击，展示相应的详细信息

这样用户可以：
- 点击错误通知查看完整的错误堆栈
- 点击警告通知了解详细的警告原因
- 点击消息通知阅读完整的消息内容
- 点击任务通知查看实时安装日志
